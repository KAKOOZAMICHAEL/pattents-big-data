# Patent Data Engineering Pipeline

A complete, modular data engineering pipeline that orchestrates the entire workflow from raw XML to database storage and analysis.

## Overview

This pipeline automates the complete data processing workflow for patent data:

```
Raw XML → Sample → Extract → Clean → Database → Analyze → Export
```

### Pipeline Stages

| Step | Module                | Input                | Output               | Status         |
| ---- | --------------------- | -------------------- | -------------------- | -------------- |
| 1    | **Sample XML**        | `ipg230103.xml`      | `sample_patents.xml` | ✓ Working      |
| 2    | **Extract Data**      | `sample_patents.xml` | `patents_data.json`  | ✓ Working      |
| 3    | **Clean & Normalize** | `patents_data.json`  | 5 CSV files          | ✓ Working      |
| 4    | **Create Database**   | Configuration        | MySQL tables         | Requires MySQL |
| 5    | **Load Data**         | CSV files            | Database records     | Requires MySQL |
| 6    | **Run Analysis**      | Database             | Query results        | Requires MySQL |
| 7    | **Export Results**    | Database             | CSV/JSON files       | Requires MySQL |

## Quick Start

### Prerequisites

```bash
pip install mysql-connector-python
pip install pandas
pip install lxml
```

### Running the Pipeline

```bash
python pipeline.py
```

The script will:

1. Create a sample of 100 patents from the full XML file
2. Extract patent data into JSON format
3. Clean and normalize the data
4. Create CSV files for database import
5. (If MySQL running) Create database and load data
6. (If MySQL running) Run analysis queries
7. (If MySQL running) Export results to CSV/JSON

## Output Files

### CSV Files (Generated by Step 3)

| File                   | Purpose                       | Columns                                    |
| ---------------------- | ----------------------------- | ------------------------------------------ |
| `patents.csv`          | Patent records                | patent_id, title, description, filing_date |
| `inventors.csv`        | Inventor records              | inventor_id, full_name, country            |
| `companies.csv`        | Company records               | company_id, company_name                   |
| `patent_inventors.csv` | Patent-Inventor relationships | patent_id, inventor_id                     |
| `patent_companies.csv` | Patent-Company relationships  | patent_id, company_id                      |

### JSON Files

| File                | Purpose                                         |
| ------------------- | ----------------------------------------------- |
| `patents_data.json` | Raw extracted patent data (intermediate format) |

### Output Directory

Results exported to `outputs/` directory when MySQL is available:

- `total_patents.csv/json` - Patent count statistics
- `inventors_per_country.csv/json` - Inventor distribution
- `top_companies.csv/json` - Top companies by patent count
- `patents_per_year.csv/json` - Patent trends over time

## Key Features

### 1. **Modular Design**

- Each step is a separate function
- Easy to modify, test, or skip steps
- Clear separation of concerns

### 2. **Comprehensive Logging**

- Timestamp for each step
- Success/failure indicators (✓/✗)
- Detailed error messages with stack traces
- Summary report at the end

### 3. **Error Handling**

- Graceful error handling with detailed reporting
- Pipeline continues even if individual steps fail
- Summary shows which steps succeeded/failed

All configuration in one place:

```python

```

## Troubleshooting

### Issue: "No such file or directory: ipg230103.xml"

**Solution:** Ensure the XML file exists in the working directory

### Issue: "ModuleNotFoundError: mysql"

**Solution:** Install MySQL connector

```bash
pip install mysql-connector-python
```

### Issue: "Can't connect to MySQL server"

**Solution:** Start MySQL server

```bash
# Windows
net start MySQL80

# macOS
mysql.server start

# Linux
sudo systemctl start mysql
```

### Issue: "KeyError" in clean_data

**Solution:** Ensure JSON file is properly formatted

- Check `patents_data.json` is valid JSON
- Verify extract step completed successfully

## Performance Tips

1. **Adjust Sample Size** - Smaller samples run faster for testing

   ```python
   'sample_count': 10  # Fast testing
   ```

2. **Parallel Processing** - For very large datasets, modify pipeline to use multiprocessing
   - Edit `clean_data.py` to process patents in parallel

3. **Database Indexes** - Add indexes in `create_db.py` for faster queries
   ```sql
   CREATE INDEX idx_patent_id ON patents(patent_id);
   CREATE INDEX idx_filing_date ON patents(filing_date);
   ```

## Extending the Pipeline

### Adding a New Step

1. Create a new function in `pipeline.py`:

```python
def step_8_custom_analysis(self):
    step_name = "Step 8: Custom Analysis"
    self.log_step(step_name)

    try:
        # Your code here
        print("Processing custom analysis...")
        self.log_success(step_name)
        return True
    except Exception as e:
        self.log_error(step_name, e)
        return False
```

2. Add to the `run()` method:

```python
def run(self):
    all_passed = all([
        # ... existing steps ...
        self.step_8_custom_analysis(),
    ])
```
